# Website monitor: an exercise

In this repository, you'll find an implementation for a simple website monitor, according to the
requirements that you can read up in the HOME_ASSIGNMENT.md file.

A candidate sent us this repository, along with an IMPLEMENTATION.md file that explains how to run it.

This implementation may be incomplete, wrong, partial, or great. It's up to you to choose what to do with it:
if you were given this repository and the home assignment instructions, and you were tasked to make it production ready,
how would you proceed?

How you choose to deal with this repository is up to you, depending on how much time you want to dedicate to it, and what you feel you should do:

* You can describe the problems you see, and how to fix them, review-style;
* You can create one or more pull requests to fix the problems you see;
* You can create PRs to completely rewrite parts of the code;
* You can create PRs to add things that you feel are missing;
* A mix of the above.
* you can even take the HOME_ASSIGNMENT.md instructions and start from scratch if you feel it's the better way to do it.


Start with the things that you deem more important first!

We also recommend that, if you choose to take a mostly-review approach, you open at least one PR as well, so that
we can evaluate your coding ability.

Once you're done, get in touch with your contact and tell them.

# A sidenote about AI usage

Using any kind of AI to complete this assignment is at your discretion; please make sure that you understand and can justify
and explain all the code that you submit. Followups may be requested during the team interview, and any assignment that
gives us the feeling of being vibe coded without actual understanding of the underlying topics will be rejected.
